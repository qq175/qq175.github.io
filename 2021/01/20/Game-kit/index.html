   public void OnHurt(Damageable damageable,DamageMessage data)
    {
        Vector3 direction = data.damagePosition - transform.position;
        direction.y = 0f;
   Vector3 localDirection = transform.InverseTransformDirection(direction);

    //是不是掉水里了 掉水里重置位置
    if (data.isResetPosition)
    {
        //失去控制
        playerInput.ReleaseControl();
        //播放死亡动画
        animator.SetTrigger("Death");
        //重置位置
        StartCoroutine(ResetPosition());
    }
    else
    {
        //播放受伤动画
        animator.SetFloat("HurtX", localDirection.x);
        animator.SetFloat("HurtY", localDirection.z);
        animator.SetTrigger("Hurt");
    }
   
}

音乐是游戏可玩的一个重要的条件

场景当中音乐的频繁使用应当设置为一个单例，进行频繁调度。

music.Play() 方法 与 sound.PlayOneShot()方法的区别，Play方法指定播放一个音乐，而PlayOneShot方法可以同时播放多种声音，所以可将二者分为背景和音效的使用

学习总结

​ 1、场景中频繁调度的类，如音乐类，应当设置为单例模式，方便其他代码的调度

​ 2、unity事件的使用，事件可以很方便的与场景中的游戏对象进行交互，可以方便的对游戏当中物体的调度。

​ 使用方法 字段名？.Invoke();

​ 3、刚体的设置，刚体应当设置为不运动的，物体的运动尽量使用代码的控制，使其不受力的影响，应情况而定

​ 4、组件Character Controller 自带的良心组件，使用时注意和刚体的区别，Move()方法不用自身的位置

​ 5、判断是否为同一层的代码使用

6、游戏当中有许多物体的方法，功能大致相似，应当考虑封装为一个类，提高代码的复用性

7、unity动画状态机的使用，如重生代码

8、预制体的使用，预制体虽然方便了同一对象的重复利用，但不同地方使用时，切记勿轻易将其覆盖原预制

​ 体，易导致其他预制使用的错误，尽量解除预制

9、销毁一个对象时，有时需要出现一个效果，可以考虑设置不一样的物体在一起，然后将当前物体隐藏，播放

​ 另一个物体。
