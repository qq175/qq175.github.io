Hexo + github 基础搭建 和 使用教程

发表于2021-01-19|更新于2021-03-01
|阅读量:

第一篇博客

首先使用hexo搭建个人博客必须准备好工具

  • 安装node.js （建议版本12.14 版本太高之后可能产生错误）

  • 安装Git

  • 注册GitHub账号

  • 安装Hexo

  • 连接Github与本地

  • 写文章和发布文章

    对应的安装可以去各自的官网进行安装下载

安装Hexo

在电脑对应位置新建一个文件夹，用来存放自己的博客文件，如D盘，进行命名并新建，这里以blog命名
D:\blog
新建完成后，在该文件夹下，右键点击Git Bash Here（前提是已经安装好了Git）
输入npm i hexo-cli -g安装Hexo ，报错可以忽视
安装完成后 可输入hexo -v 查看是否安装成功
接下来输入hexo init初始化文件夹 接下来输入npm install安装必备组件
输入hexo g生成静态网页，然后输入hexo s打开本地服务器，接着输入localhost:4000便可访问第一个博客了

总结

  • 输入 npm i hexo-cli -g 安装Hexo
  • ​ hexo -v 查看是否安装成功
  • ​ hexo init 初始化文件夹
  • ​ npm install 安装必备组件
  • ​ hexo g 生成静态网页
  • ​ hexo s 打开本地服务器

配置到github

在注册github之后 登录账号，首先找到右上角的“➕”号 点击，选择New repository进行新仓库的创建

​

首先填写Repository name信息 一定按照当前的用户名填写 username.github.io （username为你的账号）
Description选填 进行描述 后勾上Add a README file文件，点击创建仓库Create repository 由于本人已经
创建过了仓库 所以标红

配置SSH key

由于提交代码需得到github权限，但直接使用账号密码不安全，所以使用ssh key来解决本地和服务器的连接问题
$ cd ~/. ssh #检查本机已存在的ssh密钥
如果提示：No such file or directory 说明你是第一次使用git。
ssh-keygen -t rsa -C “邮件地址” 邮件地址即为你注册时使用的地址
然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，
找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，
打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key：

将刚复制的内容粘贴到key那里，title随便填，保存。

测试是否成功

$ ssh -T git@github.com
如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到

继续配置个人信息
$ git config –global user.name “liuxianan”// 你的github用户名，非昵称
$ git config –global user.email “xxx@qq.com“// 填写你的github注册邮箱

接下来打开博客根目录下的_config.yml文件，这是博客的配置文件，在这里你可以修改与博客相关的各种信息。
修改最后一行的配置：
deploy:
type: git
repository: https://github.com/qq175/qq175.github.io
branch: master
repository修改为你自己的github项目地址。

总结：

  • 先github创建仓库需注意仓库名
  • 配置SSH key
  • 配置个人信息
  • 修改博客根目录下的_config.yml文件

部署与查看

首先在博客根目录下安装一个扩展npm install hexo-deployer-git –save
接着输入 hexo d 即可导入到github 此时刷新github页面即可发现多了许多文件夹
网址栏输入 username.github.io 进行访问你的第一个博客吧！！
如果hexo d命令已经导入成功了，但是github没显示的话，注意是否有将master改为默认的Branches

总结：

  • 输入命令npm install hexo-deployer-git –save安装扩展
  • hexo d命令进行导入
  • 将github中的Branches 默认改为配置文件中的 master
  • 输入网址username.github.io进行查看 (username自身账户)
